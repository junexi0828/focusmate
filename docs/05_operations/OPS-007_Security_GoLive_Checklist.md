# OPS-007: Security Go-Live Checklist

본 문서는 운영 전 필수 보안 요건을 ISO 양식에 맞춰 체크리스트로 정리한다. 운영 중단 최소화를 전제로 **필수/권고** 항목을 구분한다.

## 1. 목적 및 범위
- **목적**: 운영 환경에서 발생 가능한 외부 공격 및 정보 유출 리스크를 최소화.
- **범위**: 인증/인가, 데이터 보호, 네트워크, 애플리케이션 보안, 운영 보안, 모니터링/대응.

## 2. 준수 기준 (ISO)
- **ISO/IEC 27001**: 정보보안 관리체계(ISMS) 요구사항.
- **ISO/IEC 27002**: 보안 통제 모범 사례.
- **ISO/IEC 27005**: 위험 관리 프로세스.

## 3. 필수 보안 체크리스트 (Go-Live 필수)
| 영역 | 항목 | 기준 | 확인 방법 |
| --- | --- | --- | --- |
| 인증/인가 | 전 API 권한 검사 | 모든 보호 리소스에 소유권/역할 검증 | 코드 리뷰/테스트 |
| 인증/인가 | 관리자 전용 API 분리 | 관리자/사용자 경로 분리 | 코드 리뷰 |
| WebSocket | 토큰 전달 안전화 | Query 금지, Header/쿠키 사용 | 코드 리뷰 |
| WebSocket | 사용자 실검증 | 토큰만 믿지 않고 DB 조회 | 코드 리뷰 |
| 비밀 관리 | SECRET/키 교체 | 기본값 금지, 키 회전 정책 | 설정 점검 |
| 데이터 보호 | DB 최소 권한 | 읽기/쓰기 최소 권한 계정 | DB 권한 점검 |
| 데이터 보호 | 백업 보호 | 암호화 + 접근 제한 | 운영 점검 |
| 입력 검증 | 서버측 검증 | 스키마/길이/형식 검증 | 코드 리뷰 |
| 파일 업로드 | 이중 검증 | 확장자+MIME 검사, 크기 제한 | 코드 리뷰 |
| 보안 헤더 | 기본 헤더 적용 | HSTS, CSP, X-Frame-Options | 구성 점검 |
| Rate Limit | 민감 API 보호 | 로그인/재설정/검색 등 제한 | 설정 점검 |
| 로그 | 민감정보 차단 | 토큰/PII 로그 금지 | 로그 샘플링 |
| 모니터링 | 이상징후 감지 | 실패 로그인, 대량 조회 알림 | 알림 테스트 |
| 취약점 관리 | 정기 스캔 | SAST/SCA 주기 운영 | 운영 정책 |

## 4. 권고 보안 체크리스트 (Go-Live 권고)
| 영역 | 항목 | 기준 | 확인 방법 |
| --- | --- | --- | --- |
| 네트워크 | WAF 적용 | OWASP Top 10 차단 룰 | 인프라 점검 |
| 네트워크 | 보안 그룹 최소화 | 최소 포트만 허용 | 인프라 점검 |
| 데이터 보호 | 데이터 마스킹 | 운영 로그/리포트 PII 마스킹 | 샘플 점검 |
| 운영 | 변경 승인 절차 | 배포 전 보안 승인 | 운영 정책 |
| 운영 | 침해 대응 리허설 | 분기 1회 모의훈련 | 운영 기록 |

## 5. 완료 기준 (Go-Live Gate)
1. 필수 체크리스트 **100% 완료**.
2. 권고 체크리스트 **70% 이상 완료**.
3. 운영 장애/성능 저하 없음.
4. 보안 이벤트 로그/알림 정상 동작.

## 6. 변경 관리
- 변경 시 **위험 평가(ISO 27005)** 수행.
- 변경 내역은 `docs/05_operations/OPS-005_Changelog.md`에 기록.
