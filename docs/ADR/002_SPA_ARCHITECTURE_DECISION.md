# ADR 002: Vite + TanStack SPA 아키텍처 채택

## 상태 (Status)
승인됨 (Accepted)

## 날짜 (Date)
2025-12-11

## 배경 (Context)
Focus Mate는 고도의 집중을 요하는 **생산성 도구**입니다. 애플리케이션의 성격상 다음이 요구됩니다:
1.  **긴 세션**: 사용자가 타이머/대시보드 화면에 몇 시간씩 머무릅니다.
2.  **높은 상호작용**: 타이머, 할 일 체크 등 UI 조작이 즉각적이어야 합니다.
3.  **앱 같은 사용감**: 문서 웹사이트가 아닌 네이티브 데스크톱/모바일 앱처럼 느껴져야 합니다.
4.  **SEO 불필요**: 핵심 기능은 로그인 후에 제공되므로 검색 엔진 인덱싱이 필수적이지 않습니다.

우리는 두 가지 고도화 옵션을 검토했습니다:
*   **옵션 A: Next.js (SSR)** - 웹사이트 및 이커머스의 업계 표준.
*   **옵션 B: Vite + TanStack Router (SPA)** - 복잡한 웹 애플리케이션(Linear, Figma 등)의 현대적 표준.

## 결정 (Decision)
우리는 **옵션 B: Vite + TanStack Router (SPA)**를 선택했습니다.

## 근거 (Rationale)
1.  **소프트웨어 본질 부합**: Focus Mate는 "사이트"가 아닌 "앱"입니다. ISO/IEC 25010의 '사용 효율성(Quality in Use)' 관점에서, HTML 생성을 위한 서버 왕복 없이 즉시 반응하는 인터페이스가 필수적입니다.
2.  **타입 안전성 (Reliability)**: TanStack Router는 100% 타입 안전한 라우팅을 제공하여, 잘못된 링크나 누락된 파라미터로 인한 런타임 오류를 획기적으로 줄여줍니다.
3.  **개발 경험 및 유지보수성**: 프론트엔드를 백엔드(FastAPI)와 완전히 분리함으로써, Next.js의 모호한 경계 없이 깔끔한 관심사 분리(Separation of Concerns)를 달성합니다.
4.  **성능 (상호작용)**: Next.js는 초기 로딩(FCP)이 빠르지만, 지속적인 사용 시의 반응성(INP)은 SPA 아키텍처가 월등히 우수합니다.

## 결과 (Consequences)
*   **긍정적**: 페이지 깜빡임 없는 매끄러운 사용자 경험 제공. OpenAPI를 통한 백엔드-프론트엔드 간의 엄격한 계약(Contract) 기반 개발.
*   **부정적**: 초기 번들 다운로드로 인해 첫 진입 속도가 SSR보다 약간 느릴 수 있음 (코드 스플리팅으로 완화 가능). 앱 페이지의 SEO가 제한됨 (본 프로젝트에서는 수용 가능).
