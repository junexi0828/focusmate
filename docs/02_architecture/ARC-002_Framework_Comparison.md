# 프레임워크 비교 및 UI/UX 영향도 분석

## 1. 프레임워크를 바꾸면 UI/UX가 달라지나요?

### UI (디자인/화면)
**결론: 바뀌지 않습니다.**
*   현재 사용 중인 **Tailwind CSS**와 **Radix UI**는 프레임워크에 종속되지 않습니다.
*   기존 컴포넌트(`src/components/...`)를 그대로 가져다 쓸 수 있으므로 시각적인 디자인은 100% 동일하게 유지됩니다.

### UX (사용자 경험/속도)
**결론: 훨씬 더 좋아집니다.**
*   **속도**: 최신 번들러(Vite)와 똑똑한 라우터(TanStack)가 코드를 자동으로 쪼개고(Split) 미리 불러와서(Preload) 매우 빠릅니다.
*   **부드러움**: SPA 아키텍처는 페이지 이동 시 "흰 화면 깜빡임"을 완전히 제거하여 네이티브 앱 같은 느낌을 줍니다.
*   **피드백**: 스켈레톤 UI나 낙관적 업데이트(Optimistic Update)를 구현하기 쉬워져서, 사용자는 앱이 멈추지 않고 반응한다고 느낍니다.

## 2. React 생태계의 다른 대안들

### 대안 A: Remix
*   **철학**: "웹 표준 준수". 브라우저의 기본 기능과 HTTP 캐시를 최대한 활용합니다.
*   **장점**: 데이터 입력(Form) 처리가 매우 강력하고 직관적입니다.
*   **단점**: Node.js 서버가 반드시 필요하여 배포/관리가 Vite보다 조금 더 복잡합니다.

### 대안 B: Vite + TanStack Router (우리의 선택)
*   **철학**: "클라이언트 중심 앱". 브라우저를 하나의 강력한 애플리케이션 플랫폼으로 취급합니다.
*   **장점**:
    *   **단순함**: 프론트엔드 전용 서버가 필요 없습니다. (S3/CDN 배포 가능)
    *   **타입 안전성**: 라우팅 관련 코드의 오타를 원천 차단하는 강력한 TypeScript 지원.
    *   **명확한 분리**: FastAPI 백엔드와 완벽하게 분리되어 개발 효율이 좋습니다.
*   **추천 대상**: "검색 노출(SEO)보다는 압도적인 반응 속도와 사용성이 중요한 서비스"

## 3. 비교 요약표

| 특징 | Next.js (보편적 표준) | Remix | Vite + TanStack Router (우리의 선택) |
| :--- | :--- | :--- | :--- |
| **핵심 가치** | 시장 표준, 방대한 자료 | 웹 표준, 데이터 관리 | **앱 같은 UX**, 타입 안전성 |
| **SEO (검색)** | 최상 (SSR) | 최상 (SSR) | 양호 (최적화 가능하나 SSR보다는 약함) |
| **러닝 커브** | 보통 | 보통 | 높음 (새로운 개념 습득 필요) |
| **인프라** | Node.js 서버 필요 | Node.js 서버 필요 | **정적 호스팅** (가볍고 저렴) |
